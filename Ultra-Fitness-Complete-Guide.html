How Your Data is Organized

The system stores information in three main categories (called "collections" in database terms):

erDiagram CUSTOMER ||--o{ ATTENDANCE : "has many" CUSTOMER ||--o{ PAYMENT : "has many" CUSTOMER { string MemberID string Name string Email string Phone number Age string Plan date ValidityDate string Photo string Password } ATTENDANCE { string MemberID date CheckInDate time CheckInTime string Status } PAYMENT { string MemberID number Amount string PlanType date PaymentDate string Method }

Data Categories Explained

1. Member Information (CUSTOMER)

What it stores: All basic information about each gym member

  • Member ID: Unique identifier (U001, U002, etc.)
  • Personal Details: Name, age, email, phone number
  • Membership Info: Current plan type, expiry date
  • Photo: Profile picture stored in cloud storage
  • Security: Encrypted password for app login

2. Attendance Records (ATTENDANCE)

What it stores: Every gym check-in for every member

  • Who: Which member checked in (linked to Member ID)
  • When: Exact date and time of check-in
  • Status: Whether membership was active, expiring, or expired at check-in time
  • Prevention: System prevents duplicate check-ins on the same day

3. Payment History (PAYMENT)

What it stores: All financial transactions

  • Transaction Details: Amount paid, payment method (Cash, UPI, Card, etc.)
  • Plan Information: Which membership plan was purchased
  • Dates: When payment was made
  • Notes: Any additional information about the payment

Technology Explained (For Non-Technical Users)

What is Node.js?

Think of it as the engine that powers the backend server. It handles all the requests from the admin portal and member app, processes them, and sends back responses.

What is MongoDB?

This is like a digital filing cabinet where all your data is stored - member information, attendance records, payment history, etc. It's hosted in the cloud, so it's accessible 24/7 from anywhere.

What is JWT (JSON Web Token)?

Think of it as a digital ID card. When you log in, the system gives you this token, and you present it with every request to prove who you are. It's more secure than traditional sessions.
